## 1. 项目背景与目标

### 1.1 课程要求

本作业来自 BUPT 计算机导论课程。OJ 版要求在 20×20 字符地图上实现贪吃蛇核心逻辑——W/A/S/D 控制移动、吃食物得分、N 步增长、撞墙/障碍物/自身死亡。OJ 版采用标准输入输出交互，程序需解析输入命令并返回地图状态。

图形化版在此基础上要求：
- 使用 EasyX 图形库替换字符显示为贴图渲染
- 添加菜单系统（欢迎界面、模式选择、设置）
- 支持多种游戏模式
- 模块化设计，代码清晰，接口便于扩展
- 最终输出一版设计说明书

### 1.2 功能清单

| 编号 | 功能 | 难度 | 说明 |
|------|------|------|------|
| 1 | 单人模式 | 基础 | 经典贪吃蛇，吃食物得分，撞墙/障碍/自身死亡 |
| 2 | AI 对战模式 | 进阶 | 与 AI 蛇同场竞技，支持低/中/高三档难度 |
| 3 | 限时挑战模式 | 进阶 | 90 秒内挑战最高分，多样道具 |
| 4 | 本地多人模式 | 进阶 | 双人同屏对抗，WASD vs 方向键，90 秒限时 |
| 5 | 多样道具系统 | 进阶 | 加分食物、加速食物、减速食物、护盾、陷阱 |
| 6 | AI 对战道具 | 进阶 | 弓箭（飞行物）、战斗护盾、尖刺陷阱、减速时钟 |

## 2. 对 OJ 版的继承与改造

### 2.1 直接继承的部分

| OJ 版逻辑 | 图形化版处理 | 说明 |
|-----------|-------------|------|
| 20×20 字符地图 | 保留 `cells[][]` 二维数组 | 改用 `CellType` 枚举代替字符 |
| 蛇身体数组存储 | 保留 `Snake.body[]` | 增加更多状态字段 |
| 移动：头插尾删 | 保留 `applySnakeMove` | 增长逻辑扩展为三种触发方式 |
| 碰撞检测（墙/障碍/自身） | 保留并扩展 | 增加蛇-蛇碰撞、道具碰撞 |
| 吃食物 +10 分 | 保留核心 | 增加速度档倍率、道具区分 |
| N 步增长 | 可配置开关 | `GameConfig.enableStepGrowth` |
| 反方向保护 | 保留 | `Common_isOpposite` 判断 |

### 2.2 替换或删除的部分

| OJ 版逻辑 | 图形化版处理 | 原因 |
|-----------|-------------|------|
| 标准输入输出交互 | 删除 | 图形化版用键盘事件 + 窗口渲染 |
| `fflush(stdout)` | 删除 | 无标准输出交互 |
| AI 自动寻路（OJ 原版） | 替换为 `ai.c` 模块 | 增强为低/中/高三档策略 |
| 系统生成食物命令 | 替换为内部随机生成 | 图形化版自主管理食物刷新 |
| 字符打印地图 | 替换为 EasyX 贴图渲染 | 见第 7 节 |
| 文本 Game Over | 替换为图形化结算画面 | 显示胜负、分数、操作按钮 |

## 4. 数据结构设计

### 4.1 全局常量

```c
#define DEFAULT_MAP_SIZE 20      // 默认地图尺寸
#define MAX_MAP_SIZE 100         // 最大地图尺寸
#define MAX_SNAKE_LEN (MAX_MAP_SIZE * MAX_MAP_SIZE)  // 蛇最大长度
#define TEXTURE_SIZE 32          // 贴图原始像素尺寸
#define MAX_ACTIVE_ARROWS 8      // 同时存在的最大弓箭数
#define ARROW_INTERVAL_MS 35     // 弓箭每格移动时间
#define TIME_LIMIT_SECONDS 90    // 限时模式时长
#define SPEED_LEVEL_MIN (-2)     // 最慢速度档
#define SPEED_LEVEL_MAX 2        // 最快速度档
```

### 4.2 核心枚举

**游戏模式 `GameMode`**：`MODE_SINGLE`（单人）、`MODE_AI_BATTLE`（AI 对战）、`MODE_TIME_CHALLENGE`（限时挑战）、`MODE_LOCAL_MULTIPLAYER`（本地多人）

**地图玩法 `MapVariant`**：`VARIANT_CLASSIC`（常规，仅食物+障碍）、`VARIANT_DIVERSE`（多样，含全部道具）

**AI 难度 `AiDifficulty`**：`AI_EASY`（保守寻路）、`AI_MEDIUM`（均衡）、`AI_HARD`（高攻击性）

**格子类型 `CellType`**：`CELL_EMPTY`、`CELL_WALL`、`CELL_OBSTACLE`、`CELL_FOOD`、`CELL_FOOD_BONUS`、`CELL_FOOD_SPEED`、`CELL_FOOD_SLOW`、`CELL_TRAP`、`CELL_SHIELD`、`CELL_BATTLE_BOW`、`CELL_BATTLE_SHIELD`、`CELL_BATTLE_SPIKE`、`CELL_BATTLE_CLOCK`

**游戏结果 `GameResult`**：`RESULT_RUNNING`、`RESULT_PLAYER_WIN`、`RESULT_AI_WIN`、`RESULT_DRAW`、`RESULT_TIME_UP`、`RESULT_PLAYER_DEAD`、`RESULT_P1_WIN`、`RESULT_P2_WIN`

### 4.3 蛇结构体 `Snake`

```c
typedef struct Snake {
    Pos body[MAX_SNAKE_LEN];    // 身体坐标数组，body[0] 为头部
    int length;                 // 当前长度
    Direction dir;              // 当前移动方向
    Direction nextDir;          // 下一帧方向（缓冲反方向保护）
    int score;                  // 得分
    int stepCount;              // N 步增长计数器
    bool alive;                 // 存活状态

    // 限时效果（毫秒计时器）
    int shieldMs;               // 护盾剩余时间
    int speedMs;                // 加速剩余时间
    int slowMs;                 // 减速剩余时间
    int dirChangeCooldownMs;    // 方向变更冷却（减速时生效）
    int slowStepCounter;        // 减速时的步数计数器

    // AI 对战专属资源
    int bowArrows;              // 弓箭数量
    int shieldCharges;          // 护盾充能次数
} Snake;
```

**设计要点**：蛇身体使用 `Pos body[]` 数组而非链表存储。原因：(1) OJ 版使用数组，继承一致性；(2) 最大长度可预计算（地图面积），数组访问 O(1)；(3) 移动时整体后移即可，无需动态分配。

### 4.4 游戏状态 `GameState`

```c
typedef struct GameState {
    CellType cells[MAX_MAP_SIZE][MAX_MAP_SIZE];  // 地图格子
    Snake player;            // 玩家蛇（单人/P1）
    Snake ai;                // AI 蛇 / 本地多人 P2
    ArrowProjectile arrows[MAX_ACTIVE_ARROWS];   // 弓箭飞行物
    GameConfig config;       // 本局配置（快照）
    GameResult result;       // 当前结果
    int speedLevel;          // 速度档 -2..2
    int moveTimerMs;         // 移动计时器
    int elapsedMs;           // 已用时间
    int remainingSeconds;    // 剩余秒数（限时/多人模式）
    unsigned int randomSeed; // 随机种子
    SoundEvent soundEvents[MAX_SOUND_EVENTS];  // 待播放音效队列
    int soundEventCount;
    char statusText[128];    // 状态文字
} GameState;
```

**设计要点**：`GameState` 是全局唯一的游戏状态结构体。地图使用 `CellType` 二维数组存储地形和道具，蛇身体不在地图数组中出现（仅存于 `Snake.body[]`），避免蛇体与道具的数据冲突。

### 4.5 游戏配置 `GameConfig`

```c
typedef struct GameConfig {
    GameMode mode;              // 游戏模式
    MapVariant variant;         // 地图玩法
    AiDifficulty aiDifficulty;  // AI 难度
    DisplayResolution resolution;  // 显示分辨率
    bool fullscreen;            // 全屏开关
    int skinId;                 // 皮肤 ID
    int mapSize;                // 地图尺寸 20/50/100
    int moveIntervalMs;         // 步进间隔（毫秒）
    int growthInterval;         // N 步增长间隔
    int timeLimitSeconds;       // 限时（秒）
    bool enableStepGrowth;      // 是否启用 N 步增长
    bool musicEnabled;          // 背景音乐开关
    bool soundEnabled;          // 音效开关
} GameConfig;
```

### 5.2 游戏逻辑模块 (`game.h` / `game.c`)

**职责**：游戏核心——地图生成、蛇移动、碰撞检测、道具效果、计分、胜负判定。这是最大的模块（约 1200 行）。

**核心接口**：

| 函数 | 说明 |
|------|------|
| `Game_makeDefaultConfig(config)` | 初始化默认配置 |
| `Game_applyModeDefaults(config, mode)` | 根据模式设置默认参数 |
| `Game_init(state, config)` | 初始化游戏状态：清地图、放蛇、生成障碍道具 |
| `Game_preparePlayerStart(state, dir)` | 根据起始方向调整蛇身体排列 |
| `Game_prepareP2Start(state, dir)` | P2 起始方向调整 |
| `Game_setPlayerDirection(state, dir)` | 设置玩家方向（含反方向保护） |
| `Game_setP2Direction(state, dir)` | 设置 P2 方向 |
| `Game_update(state, deltaMs)` | 主更新函数：计时、移动、碰撞、道具刷新 |
| `Game_isFinished(state)` | 判断游戏是否结束 |
| `Game_adjustSpeed(state, delta)` | 调整速度档（多人模式禁用） |
| `Game_playerFireArrow(state)` | 玩家射箭 |
| `Game_player2FireArrow(state)` | P2 射箭 |
| `Game_aiFireArrow(state)` | AI 射箭 |
| `Game_isInside(state, pos)` | 检查坐标是否在地图内 |
| `Game_isFood(cell)` | 判断是否为食物类型 |
| `Game_isBattleItem(cell)` | 判断是否为战斗道具 |
| `Game_isDangerCell(cell)` | 判断是否为危险格子 |
| `Game_pushSoundEvent(state, event)` | 压入音效事件 |
| `Game_consumeSoundEvents(state, out, max)` | 取出待播放音效 |

**内部实现要点**：

- **地图生成** `clearCells`：边界填充 `CELL_WALL`，内部清空。
- **道具维护** `maintainItems`：根据模式调用 `maintainNormalItems`（常规/多样）或 `maintainBattleItems`（AI 对战/多人）。每种道具维护一个目标数量，不足时在空位随机生成。大地图按比例提高目标数。随机尝试有上限，超过后切换为确定性扫描兜底，防止大地图卡死。
- **蛇移动**：每步生成 `StepPlan` 结构体，描述下一步的坐标、方向、吃到的道具、是否死亡、是否增长。`applySnakeMove` 根据计划执行：头插入新位置，如果不增长则丢弃尾部。
- **碰撞检测**：单人模式 `addSingleSnakeCollisions` 检测自身碰撞；双蛇模式 `addBattleCollisions` 额外处理头碰头（同归于尽或错身而过）、碰对方身体。
- **道具效果** `applyItemEffect`：根据吃掉的道具类型施加效果——加分、加速、减速（作用于对手）、护盾、弓箭+1、尖刺即死。轰炸事件的护盾消耗也在此处理。

#### 5.2.x 随机事件系统

**触发条件**：仅在多样模式（VARIANT_DIVERSE）下的单人、AI 对战、本地多人、限时挑战模式中生效。正常每 10 秒触发一次；本地多人模式最后 30 秒加速为每 5 秒触发；限时挑战模式间隔缩短 25%。事件持续 12 秒，轰炸每 3 秒一个子阶段（箭雨保持 2 秒）。核心函数 `updateRandomEvents` 在 `Game_update` 中每帧调用。限时挑战每渡过一次事件 +100 分。

**地图轰炸 `EVENT_BOMBARDMENT`**：
- 初始化 `initBombardment`：在地图内随机生成 3 个矩形轰炸区，总面积约占地图 1/4，每个区面积最少 1 格
- 每 3 秒爆破一次（提前 1 秒橙色预警 bombWarning），bombActive 持续 500ms 深红，12 秒共 4 次爆破
- `buildStepPlan` 检测蛇头是否在爆破区且 bombActive：有护盾则设 shieldUsed=true，无护盾则 plan->dead=true
- 渲染：轰炸区红色（RGB 200,55,40），预警时橙红（RGB 230,110,40），爆破时深红（RGB 180,30,20）；每区一个整矩形而非逐格绘制，避免掉帧

**刀光箭影 `EVENT_ARROW_STORM`**：
- 初始化 `initArrowStorm`：随机选取地图周长一半数量的边界墙格（不重复），每格预计算指向场内的 Direction
- 每 2 秒 `spawnBorderArrow` 从所有选中边界格向场内发射无主箭矢（ownerIndex=-1）
- `applyArrowHit` 中无主箭矢对碰到的任意蛇造成伤害（头部即死，身体截断）
- 渲染：发射瞬间边界格黄色方块+红色箭头指示器（borderFlashing 持续 400ms）
- 箭矢池 MAX_ACTIVE_ARROWS=64，确保 100×100 地图 50 个边界源全部可射出
- `terrainBlocksArrow` 仅阻挡墙和障碍物，道具不拦截箭矢

### 5.3 AI 决策模块 (`ai.h` / `ai.c`)

**职责**：为 AI 对战模式提供方向决策。

**核心接口**：

| 函数 | 说明 |
|------|------|
| `Ai_decideDirection(state, difficulty)` | 返回 AI 本步方向 |

**实现原理——三级难度策略**：

**低难度 `decideEasy`**：基础 BFS 寻路，目标优先级为「最近食物 > 安全空地」。不考虑道具、不主动攻击，移动偏向保守。

**中难度 `decideMedium`**：BFS 寻路基础上，对每个候选方向综合评分：
- 朝向食物的距离权重
- 道具吸引力（弓箭 > 护盾 > 时钟）
- 风险规避（尖刺、敌方蛇头前方）
- 攻击机会（有弓箭且与玩家共线时加分）

**高难度 `decideHard`**：在中难度基础上：
- 更积极地争夺道具（提高道具权重）
- 主动寻找射击线（计算与玩家的直线距离，无障碍时优先向射击位移动）
- 预判玩家移动方向，尝试截断路径
- 根据自身弓箭/护盾数量调整风险偏好

### 5.4 输入模块 (`input.h` / `input.c`)

**职责**：通过 Win32 `GetAsyncKeyState` 读取键盘状态。

**核心接口**：

| 函数 | 说明 |
|------|------|
| `Input_readPlayerDirection()` | 读取 WASD 方向（电平检测，长按可连续移动） |
| `Input_readPlayer2Direction()` | 读取方向键（↑↓←→），用于 P2 |
| `Input_readMenu(input, out)` | 读取菜单操作（边沿检测，单次触发） |

**实现原理**：
- `isKeyDown(vk)`：调用 `GetAsyncKeyState(vk) & 0x8000`，返回当前帧按键是否按下。用于方向控制——允许按住连续移动。
- `keyPressed(input, vk)`：检测按键从"未按下"到"按下"的跳变沿。用于菜单操作——确保每次按下只触发一次。
- `MenuInput` 结构体封装所有可能的菜单/游戏操作：移动、选择、确认、取消、暂停、射箭、调速。

## 6. 核心算法与实现原理

### 6.1 蛇移动算法

**StepPlan 模式**：每步移动前，先构建一个 `StepPlan` 结构体描述本步的所有后果：

```c
typedef struct StepPlan {
    Pos next;        // 下一步头部位置
    Direction dir;   // 移动方向
    CellType item;   // 目标格的物品
    bool dead;       // 是否死亡
    bool stays;      // 是否原地不动（减速时）
    bool grow;       // 是否增长
} StepPlan;
```

流程：`buildStepPlan` 计算 next 位置 → 碰撞检测 → 移动执行 → 道具效果。这种设计将"决策"和"执行"分离，碰撞检测可以在两蛇之间交叉验证后再分别执行。

### 6.2 碰撞检测

**单人模式**：头撞墙/障碍物 → 死；头撞自身（忽略尾部即将离开的格子）→ 死。

**双蛇模式**（AI 对战 / 本地多人）：
1. 两头相撞同一格 → 同归于尽（`RESULT_DRAW`）
2. 两头交叉互换 → 同归于尽
3. 头撞对方身体 → 死（考虑对方尾部是否即将离开）
4. 头撞自身 → 死

### 6.3 道具放置与维护

每步移动后调用 `maintainItems`，确保地图上始终有目标数量的各类道具。

**大地图卡顿防护**：随机尝试次数上限 = `mapSize² × 2`（50×50 时为 5000 次，100×100 时为 20000 次）。超过上限后切换为全图扫描——逐格检查空位并放置。这保证了无论地图多大、障碍物多密集，道具放置不会陷入死循环。

### 6.4 弓箭飞行物系统

**数据结构**：
```c
typedef struct ArrowProjectile {
    bool active;
    Pos pos;           // 当前位置
    Direction dir;     // 飞行方向
    int ownerIndex;    // 发射者（0=P1, 1=P2/AI）
    int moveTimerMs;   // 移动计时器
} ArrowProjectile;
```

**飞行逻辑**：弓箭每 35ms 移动一格（比蛇的 115-140ms 快 3-4 倍）。飞行过程中检测：
- 碰到障碍物或墙壁 → 消失
- 碰到敌方蛇体 → 命中。若命中头部（`hitIndex == 0`），对方直接死亡；若命中身体，从命中段开始截断，发射者 +20 分
- 对方有护盾充能 → 消耗一个充能，弓箭消失

### 6.5 大地图视口系统

20×20 地图完整显示。50×50 和 100×100 地图采用**局部视口**——以玩家蛇头为焦点，仅渲染可见范围内的格子。

视口计算流程：
1. `cellSizeForMap(render, mapSize)` → 计算单格像素尺寸（最小 18px）
2. `visibleCellsForMap(render, mapSize, cellSize)` → 计算可见格子数
3. `clampViewportStart(focus, visibleCells, mapSize)` → 计算视口左上角，确保不越界
4. 渲染循环仅遍历视口内的格子

### 6.6 分辨率与全屏适配

**普通窗口**：若选择的分辨率超出显示器工作区，自动缩小到屏幕内并保持 16:9 比例。棋盘尺寸由窗口实际可用空间推导（`boardPixelSizeForWindow`），确保侧栏不被裁切。

**全屏**：通过 Win32 API 修改 EasyX 窗口样式：
```c
// 全屏：无边框弹窗
SetWindowLongPtr(hwnd, GWL_STYLE, WS_POPUP | WS_VISIBLE);
// 窗口：可调整的叠加窗口
SetWindowLongPtr(hwnd, GWL_STYLE, WS_OVERLAPPED | WS_CAPTION | WS_SYSMENU | WS_MINIMIZEBOX);
SetWindowPos(hwnd, HWND_TOP, x, y, w, h, SWP_FRAMECHANGED);
```

### 6.7 随机数生成

使用经典的 32 位线性同余生成器（LCG）：
```c
state->randomSeed = state->randomSeed * 1103515245u + 12345u;
```
种子由 `time(NULL)` 与状态指针地址异或生成，确保每次运行的初始状态不同。

## 7. 图形渲染与资源管理

### 7.1 贴图加载流程

1. `Render_loadSkin(render, skinId)` 根据 `SKINS[]` 表获取皮肤文件夹名
2. 遍历 `TEX_COUNT` 个贴图槽位，对每个 `TextureId`：
   - 拼接路径：`assets/<folder>/<texture_name>.bmp`
   - 检查文件是否存在（`_taccess`）
   - 若存在，`loadimage` 加载到 `IMAGE` 对象
   - 若不存在，标记为未加载，使用 `FALLBACK_COLORS[textureId]` 作为纯色备选
3. 贴图尺寸为 `textureCellSize`（与当前格子的渲染像素一致），`loadimage` 时自动缩放

### 7.2 皮肤系统

```c
static const SkinInfo SKINS[] = {
    { _T("default"), _T("像素森林") },
    { _T("neon"),    _T("霓虹夜行") },
    { _T("ice"),     _T("冰原挑战") }
};
```

每个皮肤文件夹包含 16 张同名 `.bmp` 贴图。新增皮肤只需在 `assets/` 下创建文件夹并放入贴图，然后在 `SKINS[]` 数组中添加条目。

### 7.3 贴图枚举

```c
typedef enum TextureId {
    TEX_GROUND, TEX_WALL, TEX_OBSTACLE,       // 地形
    TEX_FOOD, TEX_FOOD_BONUS, TEX_FOOD_SPEED, TEX_FOOD_SLOW,  // 食物
    TEX_TRAP, TEX_SHIELD,                      // 通用道具
    TEX_BOW, TEX_BATTLE_SHIELD, TEX_SPIKE, TEX_CLOCK,  // AI 对战道具
    TEX_PLAYER_HEAD, TEX_PLAYER_BODY,          // P1 蛇身
    TEX_AI_HEAD, TEX_AI_BODY,                  // P2/AI 蛇身（颜色不同）
    TEX_COUNT
} TextureId;
```

P1 使用 `player_head/body`，P2/AI 使用 `ai_head/body`，天然实现视觉区分。

### 7.4 游戏画面布局（以 1600×900 为例）
