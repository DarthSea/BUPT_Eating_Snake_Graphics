## 六、重要实现细节

### 1. EasyX 贴图加载

当前资源以 `.bmp` 为主，兼容 EasyX 稳定加载。

`tools/generate_assets.py` 会生成 `.png` 和 `.bmp` 两种资源。

运行时主要加载：

```text
assets/<skin_name>/<texture_name>.bmp
```

### 2. 不能对 `RenderContext` 整体 `memset`

之前出现过启动闪退，原因是 `RenderContext` 内含 EasyX 的 `IMAGE` 对象，整体 `memset` 会破坏对象内部状态。

已修复：

- 不再对 `RenderContext` 执行 `memset(render, 0, sizeof(*render))`。
- 只初始化必要的标量字段。

### 3. 大地图视口

大地图不再完整缩小显示，而是：

- 计算 `cellSize`。
- 计算当前可见格子数 `visibleCells`。
- 以玩家蛇头为焦点计算 `startRow`、`startCol`。
- 只绘制视口内的格子、蛇、道具、弓箭。

关键函数：

- `cellSizeForMap`
- `visibleCellsForMap`
- `clampViewportStart`
- `isInView`

### 4. 全屏实现方式

EasyX 没有直接全屏参数，因此使用 Win32：

- `GetHWnd()` 获取 EasyX 窗口句柄。
- 全屏时设置 `WS_POPUP | WS_VISIBLE`。
- 窗口模式时恢复 `WS_OVERLAPPED | WS_CAPTION | WS_SYSMENU | WS_MINIMIZEBOX`。
- 使用 `SetWindowLongPtr` 和 `SetWindowPos` 应用样式。

关键函数：

- `resolveDisplaySize`
- `boardPixelSizeForWindow`
- `applyNativeWindowMode`
- `Render_applyDisplayMode`

### 5. 反方向输入处理

当前行为：

- 如果用户输入当前方向的反方向，则忽略该输入。
- 不会立即死亡。

### 6. 多样模式卡顿修复

随机放置不再无限循环：

- 先进行有限次数随机尝试。
- 若失败，则从地图扫描空格进行确定性放置。
